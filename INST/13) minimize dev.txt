


Почему мы отключаем минификацию в Development?
Скорость Hot Reload: В режиме разработки ты меняешь код каждые 30 секунд. 
Если Webpack будет каждый раз запускать Terser и CssMinimizer, пересборка проекта займет 5–10 секунд вместо 0.5 секунды. 
Это убивает продуктивность.
Читаемость при отладке: Если у тебя возникнет ошибка, ты хочешь видеть в консоли браузера Uncaught Error at myFunction (App.js:10), 
а не Uncaught Error at a (main.js:1). Минификация превращает код в "кашу", в которой невозможно копаться.
Source Maps: Хотя карты кода помогают дебажить сжатый код, работать с "живым", несжатым кодом всегда в разы быстрее и надежнее.



optimization: {
    // Если isDev = true (npm start), то minimize = false (сжатие выключено)
    // Если isDev = false (npm run build), то minimize = true (сжатие включено)
    minimize: !isDev, 
    
    minimizer: [
      new TerserPlugin(),
      new CssMinimizerPlugin(),
    ],
  },
