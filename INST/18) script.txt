"scripts": {
  "start": "cross-env NODE_ENV=development webpack serve",
  "build": "cross-env NODE_ENV=production webpack",
  "check": "npm run lint && npm run type-check",
  "type-check": "tsc --noEmit",
  "lint": "eslint src --ext .ts,.tsx",
  "lint:fix": "eslint src --ext .ts,.tsx --fix",
  "format": "prettier --write \"src/**/*.{ts,tsx,scss,css,json}\"",
  "clear": "rd /s /q dist" 
}


Разбор каждого скрипта:
start: Запускает webpack-dev-server. Включается Hot Reload, код собирается быстро (без минификации), и ты видишь изменения в браузере мгновенно.

build: Финальная сборка. Включаются Terser, CssMinimizer, SplitChunks. На выходе в папке dist — максимально легкие и быстрые файлы.

check: «Комбо-команда». Обычно запускается перед тем, как отправить код в репозиторий. Проверяет сразу и стилистику (ESLint), и типы (TS).

type-check: Запускает компилятор TypeScript в режиме «только проверка». Он не создает JS-файлы (--noEmit), а просто сообщает, если где-то нарушена логика типов.

lint / lint:fix: Поиск ошибок в коде. Версия с :fix автоматически исправит то, что можно (например, заменит var на let или удалит лишние пробелы).

format: Принудительное форматирование всего проекта через Prettier. Полезно, если над проектом работают несколько человек с разными настройками редакторов.

clear: Удаление папки dist. (Примечание: Команда rd /s /q — для Windows. Если ты на Mac/Linux, используй rm -rf dist).
