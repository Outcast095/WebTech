


Это критически важный этап.  
Ручной запуск npm run build после каждой запятой — это путь к выгоранию. 
Webpack Dev Server превращает разработку в удовольствие, удерживая весь проект в оперативной памяти (RAM) 
для молниеносного обновления.

1. Установка 
npm install webpack-dev-server --save-dev


2. Настройка в webpack.config.js
devServer: {
    static: {
      directory: path.join(__dirname, 'dist'), // Откуда сервер берет статику
    },
    compress: true,   // Включает gzip сжатие для скорости
    port: 3000,       // Твой локальный адрес: http://localhost:3000
    open: true,       // Автоматически откроет браузер при старте
    hot: true,        // 🔥 Включает Hot Module Replacement (HMR)
  },


  ///////////////////////////////////

  В чем магия Hot Reload (HMR)?
Без HMR: Ты меняешь цвет кнопки -> Страница полностью перезагружается -> Ты теряешь данные в формах и состояние приложения.
С HMR (hot: true): Webpack «подменяет» только измененный кусочек кода (например, CSS-стиль) прямо на лету, не перезагружая страницу целиком.


Важно от Архитектора: webpack-dev-server не создает файлы в папке dist. Он собирает бандл в оперативной памяти. 
Если ты удалишь папку dist и запустишь npm start, всё будет работать, 
но физических файлов ты не увидишь. Для генерации файлов на диске используй npm run build.


////////// 
3. настройка package.json

добавляем в раздел "scripts"

"scripts": {
    "start": "webpack serve --mode development --open", 
    или 
     "dev": "webpack serve --mode development --open",
  },
