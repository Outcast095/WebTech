
подключение картинок и шрифтов 

Есть 4 типа ресурсов:
asset/resource: Копирует файл в папку dist и выдает путь к нему (идеально для крупных картинок и шрифтов).
asset/inline: Превращает файл в Base64-строку и вставляет прямо в JS/CSS 
(хорошо для крошечных SVG или иконок в 1-2 КБ, чтобы сэкономить на HTTP-запросах).
asset/source: Импортирует содержимое файла как строку (например, если тебе нужно прочитать .txt или .svg как код).
asset (Automatic): Webpack сам решит: если файл меньше 8 КБ, он сделает его inline, если больше — resource.



//////////////////////////////////
Создаем папку src/assets/images и положим туда любую картинку.



////////////////////////////////// 
настройки в webpack.config.js

module: {
    rules: [
      // Обработка картинок
      
      {
        test: /\.(png|svg|jpg|jpeg|gif)$/i,
        type: 'asset/resource',
        generator: {
          // Куда именно в dist складывать картинки и как называть
          filename: 'assets/images/[hash][ext][query]'
        }
      },

      // Обработка шрифтов
      {
        test: /\.(woff|woff2|eot|ttf|otf)$/i,
        type: 'asset/resource',
        generator: {
          filename: 'assets/fonts/[name][ext]'
        }
      },
    ],
  }



3. этот код вставляется в index.js для того что бы протестировать 
ибо если в index.js нет операции с картинкой в конечном бандле никаких изменений не будет. 


  console.log("hello world");

// Импортируем картинку как модуль
import myImage from './assets/images/Magomed.png';  <= название изображения и путь до него

// Теперь можно использовать
const img = document.createElement('img');
img.src = myImage; // Webpack подставит правильный путь
document.body.appendChild(img);

console.log('Изображение загружено:', myImage);
