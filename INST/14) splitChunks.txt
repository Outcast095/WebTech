

splitChunks нужен для того, чтобы твой сайт загружался мгновенно при повторном посещении.

Главные причины:
Умный кэш: Библиотеки (React, Redux, Axios) весят много, но ты их обновляешь раз в полгода. Твой код весит мало, 
но ты меняешь его каждый день. Благодаря разделению, браузер один раз скачивает тяжелый «React» и сохраняет его в кэше, 
а при обновлениях докачивает только твои маленькие изменения.
Параллельная загрузка: Современные браузеры быстрее скачивают три файла по 50 КБ одновременно, чем один файл на 150 КБ.
Быстрая отрисовка: Браузер может начать обрабатывать тяжелые библиотеки, пока твой основной код еще летит по сети.


Настройка splitChunks находится внутри объекта optimization.


output: {
    path: path.resolve(__dirname, 'dist'),
    // Замени bundle.js на этот шаблон:
    filename: isDev ? '[name].js' : '[name].[contenthash].js',
    clean: true,
  },

  //////////////////////
optimization: {
  splitChunks: {
    chunks: 'all', 
  },
},
