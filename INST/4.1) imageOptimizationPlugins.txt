

Проблема: Большие картинки замедляют загрузку сайта

Решение: Плагин автоматически сжимает картинки при сборке

Что делает плагин?
1. Webpack собирает проект
2. Копирует картинки в dist (asset/resource)
3. ImageminPlugin проходит по всем картинкам в dist
4. Сжимает каждую (JPEG, PNG)
5. Опционально создает WebP версии
6. Готово! 🎉


Ключевые преимущества
1. Кэширование — повторные сборки быстрее 
2. WebP конвертация — современный формат (на 25-30% легче) 
3. Только production — не тормозит разработку 
4. Минимум зависимостей — чистый npm install


1. установка 
npm install image-minimizer-webpack-plugin sharp --save-dev

2. настройка файла webpack.config.js


import ImageMinimizerPlugin from 'image-minimizer-webpack-plugin';

export default {
  // ... остальные настройки (entry, output, и т.д.)



optimization: {
    minimizer: [
      new ImageMinimizerPlugin({
        minimizer: {
          implementation: ImageMinimizerPlugin.sharpMinify,
          options: {
            encodeOptions: {
              jpeg: { quality: 75, progressive: true },
              png: { quality: 80, palette: true },
              webp: { quality: 75 },
              avif: { quality: 50 },
            },
          },
        },
        
        generator: [
          {
            preset: "webp",
            implementation: ImageMinimizerPlugin.sharpGenerate,
            options: {
              encodeOptions: {
                webp: { quality: 75 },
              },
            },
          },
        ],
      }),
    ],
  },


4. Важный момент про Asset Modules
Чтобы Webpack вообще "увидел" твои картинки и передал их плагину Sharp, в module.rules должна быть такая настройка:

module: {
  rules: [
    {
      test: /\.(png|jpe?g|gif|svg)$/i,
      type: 'asset/resource', // Копирует файл в dist и оптимизирует его
    },
  ],
},

console.log("hello world");

// Импортируем картинку как модуль
import myImage from './assets/images/spider.png'

// Теперь можно использовать
const img = document.createElement('img');
img.src = myImage; // Webpack подставит правильный путь
document.body.appendChild(img);

console.log('Изображение загружено:', myImage);


// Обычный импорт (даст PNG)
import spiderPng from './assets/images/spider.png';

// Импорт через генератор (даст WebP)
import spiderWebp from './assets/images/spider.png?as=webp';

console.log('Путь к PNG:', spiderPng);
console.log('Путь к WebP:', spiderWebp);
