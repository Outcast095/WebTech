

Когда проект растет, импорты превращаются в ад: ../../../components/Button. 
С Aliases (псевдонимами) ты просто пишешь @/components/Button.
Так как мы используем TypeScript, нам нужно настроить это в двух местах, чтобы и Webpack понимал, 
куда идти, и TypeScript не ругался на «неизвестный модуль».


1. Настройка в tsconfig.json

{
  "compilerOptions": {
    "baseUrl": ".", 
    "paths": {
      "@/*": ["src/*"] 
    }
  }
}

2. Настройка в webpack.config.js

import path from 'path';

// ... (твой остальной конфиг)

export default {
  // ...
  resolve: {
    extensions: ['.tsx', '.ts', '.jsx', '.js'],
    alias: {
      // Ключ '@' должен совпадать с тем, что в tsconfig
      '@': path.resolve(__dirname, 'src'),
    },
  },
}


Как это использовать в коде?
Теперь в любом файле (неважно, как глубоко он запрятан) ты можешь писать так:

// Вместо этого:
// import { Button } from '../../../../components/UI/Button';

// Пишем так:
import { Button } from '@/components/UI/Button';
import { useAuth } from '@/hooks/useAuth';
