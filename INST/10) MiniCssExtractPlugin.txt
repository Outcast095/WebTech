

Если максимально кратко, MiniCssExtractPlugin делает две вещи:

Выселяет стили из JS: По умолчанию Webpack заталкивает весь CSS внутрь JavaScript-бандла. 
Этот плагин достает их оттуда и создает отдельные .css файлы.

Ускоряет загрузку: Когда стили лежат в отдельном файле, браузер скачивает и обрабатывает их параллельно с JavaScript,
а не ждет, пока загрузится и выполнится огромный JS-файл.


///////////////////////////////////////
 1. установка 
npm install --save-dev mini-css-extract-plugin


///////////////////////////////////////
 2. настройка 
 В режиме isDev мы оставляем style-loader, потому что MiniCssExtractPlugin на данный момент не поддерживает 
 Hot Module Replacement (HMR) так же идеально и быстро, как это делает простой style-loader.


 импортируем в webpack.config.js

 import MiniCssExtractPlugin from 'mini-css-extract-plugin';

 в разделе  rules: [


     // 2. Стили: SASS -> PostCSS (автопрефиксы) -> CSS -> В JS или отдельный файл

     // 2. Определяем режим сборки (development или production) до  export default 
          const isDev = process.env.NODE_ENV !== 'production';

      plugins: [
          
          // ОБЯЗАТЕЛЬНО ДОБАВЬ ЭТО:
          new MiniCssExtractPlugin({
            filename: 'assets/styles/[name].[contenthash].css' // Имя и путь для CSS файла
          })
        ],


      {
        test: /\.(s[ac]ss|css)$/i,
        use: [
          // В dev-режиме вставляем стили в <style>, в prod — выносим в .css файл для скорости
          isDev ? 'style-loader' : MiniCssExtractPlugin.loader,                              <=====  подключен только в dev режиме 
          'css-loader',
          {
            loader: 'postcss-loader',
            options: {
              postcssOptions: { plugins: [['autoprefixer', {}]] },
            },
          },
          'sass-loader', // Компилирует SCSS/SASS в CSS
        ],
      },

      
 ]
